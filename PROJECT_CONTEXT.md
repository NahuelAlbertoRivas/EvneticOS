# Project Context

## Product Identity

**Product:** Evnetic OS

**Category:** Operating System for Autonomous Machines

**AI Agent:** Kairo

**Development Stage:** Minimum Commercial Product (MCP)

---

# Strategic Vision

Evnetic OS is being designed as a software platform for autonomous machines.

The initial physical robot is not the final product.

It is the first execution environment of the platform.

The long-term objective is to support multiple autonomous devices through the same operating system architecture.

---

# MCP Objective

The first version of Evnetic OS must demonstrate:

## Autonomous Device Management

Ability to:

* register devices.
* authenticate machines.
* monitor status.
* manage capabilities.

---

## Mission-Based Operation

Users should interact through objectives instead of low-level commands.

Example:

```
User:

Inspect the warehouse


System:

Create mission

Plan execution

Assign device

Execute capabilities

Generate report
```

---

## Multi Interface Experience

Initial interfaces:

* Web dashboard.
* Android application.
* Voice interaction.

All interfaces consume the same platform APIs.

---

# Intelligence Model

Kairo provides:

* natural language understanding.

# Development Philosophy

* architectural correctness.
* maintainability.
