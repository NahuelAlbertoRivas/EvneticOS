# Mission Domain

## 1. Overview

The Mission Domain represents autonomous objectives executed by devices managed by Evnetic OS.

A mission transforms human intent into controlled execution.

Example:

User:

> "Inspect the warehouse"

Becomes:

```
Mission

Goal:
inspect warehouse

Constraints:
20 minutes

Required capabilities:
navigation
vision
analysis
```

---


Possible states:

```
FAILED

CANCELLED

PAUSED
```

---

# 4. Mission Components

## Goal

What must be achieved.

Example:

```
Detect anomalies
```

---

## Constraints

Rules that limit execution.

Examples:

```
Maximum duration

Allowed area

Safety restrictions
```

---

## Plan

Sequence generated by planners.

Example:

```
1. Navigate

2. Capture images

3. Analyze

4. Report
```

---

# 5. Relationship With Kairo

Kairo does not execute missions directly.

Flow:

```
User Intent

↓

Kairo

↓

Mission Creation

↓

Mission Service

↓

Device Execution
```

---

# 6. Safety

Every mission must pass:

* permission validation.
* capability validation.
* safety constraints.

---

# 7. Boundaries

Mission Domain owns:

✓ objectives
✓ planning state
✓ execution lifecycle
✓ mission history

Does not own:

✗ robot control
✗ hardware communication
✗ AI provider integration
