# Billing Domain

## 1. Overview

The Billing Domain defines the commercial model of Evnetic OS.

Its responsibility is managing subscriptions, usage limits, resource consumption and commercial entitlements across organizations.

Evnetic OS is designed as a platform where autonomous machines, intelligence services and data infrastructure operate together.

Therefore, value generation is not limited to hardware acquisition.

The platform provides ongoing services:

* Kairo intelligence.
* cloud processing.
* data storage.
* historical analysis.
* advanced capabilities.
* integrations.
* fleet management.

---

# 2. Core Principle

Evnetic OS separates:

```text
Hardware Ownership

+

Platform Subscription

+

Resource Consumption
```

A customer may own a device but subscribe to different levels of platform capabilities.

Example:

```text
Device

Rover 001


Subscription

Evnetic Professional


Capabilities

✓ Kairo Advanced
✓ Cloud Memory
✓ Historical Analytics
✓ Remote Missions
```

---

# 3. Commercial Model Layers

The billing model is divided into three layers.

---

# Layer 1 — Platform Subscription

Defines the base capabilities available to an organization.

Example plans:

```text
Free

Professional

Business

Enterprise
```

---

## Free

Designed for:

* evaluation.
* personal projects.
* development.

Possible limits:

* number of devices.
* Kairo interactions.
* storage.
* history retention.

---

## Professional

Designed for:

* individual users.
* small businesses.

Includes:

* advanced Kairo usage.
* larger storage.
* remote access.
* analytics.

---

## Business

Designed for:

* operational environments.

Includes:

* multiple devices.
* teams.
* permissions.
* longer retention.
* integrations.

---

## Enterprise

Designed for:

* industrial customers.

Includes:

* custom limits.
* dedicated infrastructure.
* SLA.
* advanced security.

---

# 4. Kairo Consumption Model

Kairo introduces variable operational costs.

The system must not couple user experience directly to an AI provider.

Architecture:

```text
Kairo

↓

AI Provider Layer

↓

External Models
```

Possible providers:

* OpenAI.
* OpenRouter.
* Anthropic.
* Local models.

# 7. Storage Consumption Model

Storage is another variable cost.

Evnetic OS manages:

* images.
* videos.
* maps.
* telemetry.
* logs.
* AI memory.
* mission history.

---

# 8. Storage Categories

## Mandatory System Data

Required for operation.

Examples:

* device identity.
* security certificates.
* configuration.
* mission state.

This data cannot be deleted while required.

---

## User Data

Owned by customers.

Examples:

* recordings.
* captured images.
* reports.
* custom knowledge.

---

## AI Memory Data

Generated by Kairo.

Examples:

* learned preferences.
* historical context.
* semantic knowledge.

---

# 15. Relationship With Identity

Billing belongs to organizations, not users.

Example:

```text
Company A

|

Subscription

|

Users

|

Devices
```

This avoids individual ownership problems.

---

# 16. Relationship With Kairo

Kairo should know:

* available capabilities.
* usage limits.
* enabled models.

Kairo should not know billing implementation.

Incorrect:

```text
Kairo

↓

Stripe API
```

Correct:

```text
Kairo

↓

Entitlement Service

↓

Billing Domain
```

# 18. Security Considerations

Billing data requires:

* audit logs.
* immutable transactions.
* access restrictions.
* encryption.

---

# 19. Domain Responsibilities

Billing owns:

✓ subscriptions
✓ plans
✓ quotas
✓ usage tracking
✓ entitlements
✓ commercial limits

Billing does not own:

✗ AI execution
✗ device operation
✗ user identity
✗ data processing
